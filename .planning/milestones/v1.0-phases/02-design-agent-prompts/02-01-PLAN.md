Use the shared XML structure: `<purpose>`, `<context>`, `<rules>`, `<output_format>`.

**`<purpose>` section:**
- Stack conventions agent. Runs once at init.
- Creates `.planning/STACK.md` -- translation layer for design agents.
- "Rosetta Stone": spacing, color, typography, motion principles are universal; implementation syntax varies by framework.

**`<context>` section:**
- Read `.planning/DESIGN.md > Solution Space > Tech Stack` for framework, styling approach, key libraries.

**`<rules>` section:**
Generate translation recipes for ONLY these four design dimensions:
1. **Spacing**: 8pt grid values mapped to framework syntax (e.g., Tailwind: p-2/p-4/p-6/p-8; CSS Modules: padding: 8px/16px/24px/32px; vanilla CSS: custom properties)
2. **Color**: semantic color tokens mapped to framework syntax (e.g., Tailwind config: colors.primary; CSS: --color-primary)
3. **Typography**: type scale values mapped to framework syntax (e.g., Tailwind: text-sm/text-base/text-lg; CSS: font-size custom properties)
4. **Motion**: animation patterns mapped to framework/library syntax (e.g., Framer Motion: motion.div with initial/animate; GSAP: gsap.to; CSS: @keyframes + transition)

Include a fallback section for non-UI stacks: "No UI framework detected. Design agents will use generic CSS/HTML conventions."

Do NOT include: linting, testing, file structure, import conventions, git rules, TypeScript patterns. Scope is ONLY design-relevant translations.

**`<output_format>` section:**

<task type="auto">
  <name>Task 2: Create UI design agent prompt</name>
  <files>workflows/design/ui-design.md</files>
  <action>
Write `workflows/design/ui-design.md` -- the visual design specification agent.

This agent is spawned per UI phase by the Phase 3 orchestrator. It reads DESIGN.md and STACK.md, produces structured visual design specifications, and returns them to the orchestrator (does NOT write files).

Use the shared XML structure: `<purpose>`, `<context>`, `<rules>`, `<output_format>`.

**`<purpose>` section:**
- UI design agent. Produces visual design specifications for a phase's UI components.
- Enforces concrete scales and rules -- not suggestions. Opinionated specialist.
- Domain: spacing, color, typography, elevation, border-radius, component states.
- Does NOT cover: UX behavior (that's ux-design agent) or animation (that's motion-design agent).

**`<context>` section:**
- Read `.planning/DESIGN.md` -- Brand Identity (Color Mood, Typography Feel, Visual Density) and Emotional Core (personality constraining visual choices).
- Read `.planning/STACK.md` -- framework-specific syntax for recommendations.
- Read phase context provided by orchestrator (what's being built this phase).
- If Brand Identity fields are missing: default to neutral/balanced/geometric.

**`<rules>` section -- organized by design dimension, NOT component type:**

1. **Spacing (8pt Grid):**
   - All spacing multiples of 8px. 4px for fine-tuning only.
   - Scale: 4, 8, 16, 24, 32, 48, 64px.
   - Rule: internal spacing <= external spacing.
   - Conditional recipe keyed to DESIGN.md > Visual Density:
     - Spacious: favor 24-48px between sections, 16-24px between elements
     - Balanced (default): 16-32px sections, 8-16px elements
     - Dense: 8-16px sections, 4-8px elements
     - If absent: default to balanced

2. **Color (60-30-10):**
   - 60% dominant (background), 30% secondary (surfaces), 10% accent (interactive).
   - Max 3 hues + neutrals. No pure black (#000) or white (#FFF).
   - Contrast minimum: 4.5:1 body text, 3:1 large text (WCAG AA).
   - Conditional recipe keyed to DESIGN.md > Color Mood:
     - Warm: primary hue 20-45 (orange-amber), warm neutrals (e.g., #78716c base)
     - Cool: primary hue 200-230 (blue-slate), cool neutrals (e.g., #64748b base)
     - Neutral (default): achromatic primary, true neutrals
     - If absent: default to neutral

3. **Typography:**
   - Max 2 typefaces. Ratio-based scale.
   - Conditional recipe keyed to DESIGN.md > Typography Feel:
     - Geometric: Inter, Helvetica Neue family. Scale ratio 1.200
     - Humanist: Source Sans, Lato family. Scale ratio 1.250
     - Monospace: JetBrains Mono, Fira family. Scale ratio 1.125
     - If absent: default to geometric, ratio 1.200
   - Line height: 1.5 body, 1.2-1.3 headings. Never below 1.2.

4. **Component States:**
   - Every interactive element MUST have: default, hover, focus, active, disabled (opacity 40%), loading, error, empty.
   - Focus: visible ring (never remove outline without replacement).
   - Buttons + inputs share height scale (32, 36, 40, 48px).
   - Color is never the sole conveyor of meaning.

5. **Accessibility:**
   - 4.5:1 contrast body text, 3:1 large text.
   - Semantic HTML first. aria-label on icon-only buttons.
   - Focus order matches visual reading order.

**CRITICAL -- Stack Agnosticism (R2.5):**
- Do NOT mention any specific framework, library, or tool by name (React, Vue, Tailwind, Framer Motion, Angular, Svelte, etc.).
- State principles in framework-neutral terms: "16px spacing" not "p-4".
- Reference "see STACK.md for framework syntax" when implementation-specific guidance is needed.

**`<output_format>` section:**
Structured markdown with sections: Spacing System, Color System, Typography System, Component States, Accessibility Requirements.

**Token budget:** Target ~1200-1500 tokens. Prioritize principles with concrete defaults. Use conditional recipes for DESIGN.md adaptation. Reference STACK.md instead of inlining framework examples.
