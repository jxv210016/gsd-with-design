<task type="auto">
  <name>Task 1: Create install.sh POSIX sh overlay installer</name>
  <files>install.sh</files>
  <action>
Create `install.sh` at the project repo root. Must use `#!/bin/sh` shebang and be strictly POSIX sh compliant (no bashisms).

**POSIX sh constraints (R6.6):**
- Use `[ ]` not `[[ ]]` for tests
- No arrays -- use space-separated strings or here-documents for file lists
- No `local` keyword -- use subshells or unique variable prefixes for scoping
- No `$(< file)` -- use `$(cat file)`
- No brace expansion `{a..z}`
- No `function` keyword -- use `name() { ... }` syntax

**Script structure:**

1. **Header:** Shebang `#!/bin/sh`, `set -eu` (exit on error, undefined vars). Print banner: "GSD with Design -- Overlay Installer v1.0.0"

2. **Checksum function:** `compute_checksum()` that tries `shasum -a 256` first, falls back to `sha256sum`, errors if neither found. Extract just the hash (first field).

3. **Source directory detection:** The script runs from the repo root. Verify required source files exist by checking for `.claude/commands/gsd/design-thinking.md` and `workflows/design/ui-design.md`. If missing, error: "Run this script from the gsd-with-design repository root."

4. **GSD installation detection (R6.5):** Search for existing GSD installations by checking for `get-shit-done/VERSION` file in:
   - Local dirs: `./.claude`, `./.config/opencode`, `./.opencode`, `./.gemini`
   - Global dirs: `$HOME/.claude`, `$HOME/.config/opencode`, `$HOME/.opencode`, `$HOME/.gemini`

   Collect all found installations. If zero found: error "No GSD installation found. Install GSD first: npx get-shit-done-cc@latest". If one found: use it. If multiple found: list them numbered and prompt user to choose with `read` (default to first found).

5. **Existing design installation check:** If `design-version.json` exists at the target, read it. Compare checksums of installed files against stored checksums. If any mismatch, warn: "Customized design files detected. Back up? [Y/n]" -- if yes, copy customized files to `{INSTALL}/get-shit-done/design-backup-{timestamp}/`.

6. **Path rewriting for command shims:** The patched command shims in `.claude/commands/gsd/` contain `@/Users/jayvanam/.claude/` paths. Before copying, use `sed` to replace this developer path with the installing user's actual install directory path. Process each command shim through: `sed "s|@/Users/jayvanam/.claude/|@${INSTALL_DIR}/|g"`. Apply this to: new-project.md, discuss-phase.md, plan-phase.md, update.md.

   Also process workflow files that may contain the developer path: workflows/new-project.md, workflows/discuss-phase.md, workflows/plan-phase.md. Apply the same sed replacement.

7. **File copy:** Create `get-shit-done/workflows/design/` directory at install target if it does not exist. Copy files using `cp -p` (preserve permissions):
   - Design-only command shims: copy directly (no path rewriting needed for design-thinking.md, design-ui.md, design-stack.md)
   - Patched command shims: copy through sed path rewriting (new-project.md, discuss-phase.md, plan-phase.md, update.md)
   - Design workflow files: copy directly to `get-shit-done/workflows/design/`
   - Patched workflow files: copy through sed path rewriting to `get-shit-done/workflows/`

8. **design-version.json generation:** After all files are copied, compute SHA-256 checksums of all 16 installed files. Read GSD base version from `get-shit-done/VERSION`. Generate `design-version.json` at `{INSTALL}/get-shit-done/design-version.json` using printf (no jq dependency). Use the schema from the interfaces block.

9. **Verification:** Count installed files, verify all 16 exist. Print summary: files installed count, install location, GSD base version.

10. **Success message:** "GSD with Design installed successfully! Restart Claude Code to use design commands."

Make the file executable (`chmod +x` will be done in verify step).
