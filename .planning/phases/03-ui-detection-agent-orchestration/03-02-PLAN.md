Create `workflows/design/orchestrate-design.md` using the shared `<purpose>/<context>/<rules>/<output_format>` XML structure. This is the orchestrator workflow that discuss-phase calls after UI detection returns true.

The file must contain these sections:

**purpose:** This is the design agent orchestrator. It receives a confirmed UI phase, spawns design agents, collects their outputs, resolves conflicts, and writes `{phase}-UI.md`. It is called by discuss-phase after CONTEXT.md is written and UI detection returns IS_UI=true.

**context:** Specify the inputs the orchestrator receives:
- Phase number and name (from discuss-phase caller)
- Phase goal (from ROADMAP.md)
- Phase directory path (e.g., `.planning/phases/XX-name/`)
- CONTEXT.md path for the phase
- REFRESH_STACK flag (from ui-detection, if `<!-- design-refresh-stack -->` marker was found)

**rules:** The orchestration algorithm in this exact order:

**Step 1 — Stack-conventions gate (init-once):**
- Check if `.planning/STACK.md` exists
- If it does NOT exist OR REFRESH_STACK=true:
  - Spawn stack-conventions agent: `Task(description="Stack conventions: create STACK.md", prompt="Read workflows/design/stack-conventions.md for your instructions. <files_to_read> .planning/DESIGN.md </files_to_read> Write .planning/STACK.md using the Write tool.")`
  - Wait for completion
  - Verify `.planning/STACK.md` was created. If not, warn and abort design orchestration
- If it exists and REFRESH_STACK=false: log "Using existing .planning/STACK.md" and proceed

**Step 2 — Parallel design agent spawning:**
- Spawn all 3 design agents simultaneously using Task() with run_in_background=true
- Each agent gets the same 4 inputs:
  1. `.planning/DESIGN.md` (brand direction)
  2. `.planning/STACK.md` (framework conventions)
  3. `{phase_dir}/{phase}-CONTEXT.md` (user decisions for this phase)
  4. Phase goal text from ROADMAP.md

- Agent spawn prompts (include all 3 with their specific workflow file references):

  UI Design agent:
  ```
  Task(run_in_background=true, description="UI design: Phase {X}", prompt="
  Read workflows/design/ui-design.md for your complete instructions.
  <files_to_read>
  - .planning/DESIGN.md
  - .planning/STACK.md
  - {context_path}
  </files_to_read>
  <phase_context>Phase {X}: {name} — Goal: {goal}</phase_context>
  Return your output as structured markdown sections. Do NOT write any files.")
  ```

  UX Design agent: same pattern, references workflows/design/ux-design.md
  Motion Design agent: same pattern, references workflows/design/motion-design.md

- Wait for all 3 agents to complete. Collect their outputs.

**Step 3 — Failure handling and retry:**
- Track which agents completed successfully and which failed
- If 1-2 agents failed: retry each failed agent ONCE (same prompt). If still fails after retry, proceed with available outputs
- If all 3 agents failed: retry ALL agents once. If still all failing, skip {phase}-UI.md creation entirely and warn: "All design agents failed. Continuing without design guidance for this phase."
- Record agents_completed and agents_failed lists for frontmatter

**Step 4 — Synthesis into {phase}-UI.md:**
- Concatenate agent outputs with clear section headers (## UI Design, ## UX Design, ## Motion Design)
- Generate a "Design Constraints (Quick Reference)" summary section at the top with 5-8 key constraints extracted from agent outputs (e.g., "8pt spacing grid", "Max 5 options per choice point", "200ms transitions", "4.5:1 contrast minimum", "prefers-reduced-motion: opacity-only fallbacks")
- Apply conflict resolution hierarchy to detect and resolve contradictions:
  1. UX rules > Visual design rules (usability wins over aesthetics)
  2. Accessibility rules > Motion rules (a11y is non-negotiable)
  3. Brand direction (DESIGN.md) = tiebreaker when rules are equivalent
- Known conflict patterns to scan for:
  - Touch target size vs component size specifications
  - Animation duration vs feedback timing
  - Color contrast vs decorative opacity
  - Information density vs cognitive load limits
- Record each resolution inline: "(Resolved: UX recommendation took priority over motion preference)"
- Add a ## Conflict Resolutions section at the bottom listing all detected conflicts

**Step 5 — Write {phase}-UI.md:**
- Use the Write tool to create `{phase_dir}/{phase}-UI.md` with this structure:

```markdown
---
phase: {phase_number}
generated: {date}
agents_completed: [list]
agents_failed: [list or empty]
---

# Phase {X}: {Name} - Design Guidance

## Design Constraints (Quick Reference)
- {5-8 key constraints}

## UI Design
{ui-design agent output verbatim}

## UX Design
{ux-design agent output verbatim}

## Motion Design
{motion-design agent output verbatim}

## Conflict Resolutions
{List of detected conflicts and resolutions, or "No conflicts detected."}
```

- If any agent failed, omit its section and add a note: "Note: {agent-name} guidance not available for this phase. See agents_failed in frontmatter."

**output_format:** After writing {phase}-UI.md, report:
- Which agents completed successfully
- Which agents failed (if any)
- How many conflicts were resolved
- Path to the created {phase}-UI.md file

Keep the file concise — target ~150-180 lines. Use clear step-by-step instructions. This is a prompt for Claude (the discuss-phase orchestrator), not executable code. Follow the style of existing workflows/design/*.md files.
