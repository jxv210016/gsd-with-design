Create `.claude/commands/gsd/design-ui.md` -- a read-only slash command that outputs a consolidated UI/UX/motion quick-reference card.

**Frontmatter:**
```yaml
---
name: gsd:design-ui
description: Quick reference for UI/UX/motion craft standards
argument-hint: ""
allowed-tools:
  - Read
---
```

**Structure:** Follow the `<objective>` / `<process>` pattern from design-thinking.md but simpler (no interactive steps).

**Objective section:** State this is a read-only reference command. Explicitly instruct: "Output ONLY the reference content. Do NOT add project-specific analysis, next-step suggestions, or commentary beyond the reference."

**Process section with 2 steps:**

Step 1 -- Load project context:
- Read `.planning/DESIGN.md`. If it exists, extract Brand Identity (Color Mood, Typography Feel, Visual Density) and Emotional Core (Primary Emotional Statement, Supporting Attributes).
- If DESIGN.md does not exist: note "No DESIGN.md found. Showing generic defaults. Run `/gsd:design-thinking` to personalize these standards."
- Read the three design agent workflow files: `workflows/design/ui-design.md`, `workflows/design/ux-design.md`, `workflows/design/motion-design.md`.

Step 2 -- Output the reference card with these exact sections (using tables and checklists, NOT prose):

**"Your Project" section:** Color Mood, Typography Feel, Visual Density, Emotional Core (from DESIGN.md or "not set (default)" placeholders).

**"Spacing (8pt Grid)" table:** Token | Value | Use -- with xs(4px), sm(8px), md(16px), lg(24px), xl(32px), 2xl(48px), 3xl(64px). Add a note about density interpretation from DESIGN.md Visual Density.

**"Color (60-30-10)" section:** Distribution rule table (60% dominant/background, 30% secondary/surfaces, 10% accent/interactive). Constraints list: max 3 hues, no pure black/white, contrast minimums (4.5:1 body, 3:1 large text).

**"Typography Scale" section:** Max 2 typefaces. Ratio based on Typography Feel (geometric=1.200, humanist=1.250, monospace=1.125). Line-height: 1.5 body, 1.2-1.3 headings. Weights: 400 body, 600 headings.

**"Component States Checklist":** All 8 states as a checklist: Default, Hover, Focus, Active, Disabled, Loading, Error, Empty. Height scale: 32/36/40/48px.

**"UX Rules" section:** Hick's Law (5-7 options max), one primary action per view, feedback timing (100ms min, 2s+ needs progress), error format (what+why+action), honest design checklist (pricing visible, cancellation discoverable, no confirmshaming, no fake scarcity, toggles unambiguous).

**"Motion Defaults" table:** Interaction Type | Duration | Easing -- micro(100-150ms), dropdowns(150-200ms), panels(250-350ms), pages(300-500ms). Enter=ease-out, exit=ease-in. Safe properties: opacity, transform, filter only. Stagger: first 4-6 items, 0.05s delay.

**"Accessibility" section:** 4.5:1 body text contrast, 3:1 large text, 44x44px touch targets, visible focus rings, prefers-reduced-motion ALWAYS respected.

Keep the entire output scannable -- lookup tables and checklists only, no explanatory prose paragraphs.

**Process section with 2 steps:**

Step 1 -- Load project context:
- Read `.planning/DESIGN.md`. If it exists, extract Solution Space > Tech Stack (Framework, Styling, Key Libraries).

Step 2 -- Output the reference card with these exact sections:

**"Detected Stack" section:** Framework, Styling approach, Key Libraries (from DESIGN.md or "not detected").

**"Spacing Recipes" section:** If STACK.md exists, output its Spacing section (framework-specific syntax for the 8pt grid scale). If not, output the generic 8pt grid values (4, 8, 16, 24, 32, 48, 64px) as CSS custom properties: `--space-xs: 4px` through `--space-3xl: 64px`.

**"Color Recipes" section:** If STACK.md exists, output its Color section (framework-specific token syntax). If not, output generic CSS custom properties for semantic color tokens: `--color-primary`, `--color-secondary`, `--color-accent`, `--color-background`, `--color-surface`, `--color-text`, `--color-muted`, `--color-border`, `--color-error`, `--color-success`.

**"Typography Recipes" section:** If STACK.md exists, output its Typography section. If not, output generic CSS with `--font-family-heading`, `--font-family-body`, and the type scale as custom properties.

**"Motion Recipes" section:** If STACK.md exists, output its Motion section (framework/library-specific animation syntax). If not, output generic CSS: `--duration-fast: 150ms`, `--duration-normal: 250ms`, `--duration-slow: 400ms`, easing values, and a `prefers-reduced-motion` media query template.

Keep output concise -- show the pattern once with one example value per recipe. Tables for token mappings, code blocks for syntax examples.
